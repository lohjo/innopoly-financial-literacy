# Output Requirements

Write as if this is an internal design specification authored by a team from Brilliant.org, Duolingo, OpenAI, and Apple preparing for implementation by senior frontend engineers, AI engineers, conversation designers, learning scientists, UX researchers, and motion designers.

Do **not** produce mock marketing copy or superficial feature lists. Instead, justify every design decision through first principles: cognitive science, active recall, desirable difficulty, scaffolding, conversational pedagogy, self-determination theory, multimedia learning, progressive disclosure, and behavioral economics.

For every proposed interaction, explain:

* The learner's cognitive state before and after the interaction.
* Why the interaction is superior to a traditional chatbot or static lesson.
* How it contributes to long-term financial decision-making.
* How it can be implemented as reusable frontend primitives.
* How AI personalizes it over time without compromising trust or pedagogical integrity.

The final output should be detailed enough that a senior engineering team could decompose it into epics, user stories, components, and implementation tasks with minimal additional product clarification.
